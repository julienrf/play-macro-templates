We can do better than just dynamically resolving the template of a page: we can also generate a navigation menu from the list of pages. A basic menu could be generated by iterating over the page map keys:

@@for(path <- pages.keys) {
    <a href="@@routes.Application.page(path)">path</a>
}

However, that would be better to show a nice page title in the link text, rather than the page path. We can solve this problem by using the application’s i18n messages:

@@for(path <- pages.key) {
  <a href="@@routes.Application.page(path)">@@Messages(s"$path.title")</a>
}

Then in our conf/messages file we can define the title of each page:

foo.title=Foo’s page title

Actually, there is yet another problem with this menu generation system: we can not control the order of the pages in the menu. We can solve it by using a kind of naming convention for the pages. For instance, in this Web site example, page names that are prefixed by _pos_, where pos is a number, are displayed in the menu and sorted by their pos value. For a blog you could use another convention, based on a format similar to a date, e.g. _yyyy.mm.dd_, to sort your articles by date.

}
